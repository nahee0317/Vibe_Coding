# VIBE CODING 프레젠테이션 사용 설명서

> **주제:** AI 패러다임의 전환 — 바이브 코딩(Vibe Coding)
> **제작:** 황지원 | **슬라이드 수:** 총 15장

---

## 📌 개요

이 프레젠테이션은 순수 HTML · CSS · JavaScript로 제작된 **웹 기반 풀스크린 슬라이드쇼**입니다.  
별도의 설치 프로그램 없이 브라우저 하나만으로 실행되며, PowerPoint나 Keynote 없이도 세련된 발표 환경을 제공합니다.  
다크 테마의 SF 스타일 디자인, 파티클 배경 애니메이션, 글래스모피즘 카드 등 시각적 완성도를 갖춘 형태로 제작되었습니다.

---

## 🚀 실행 방법

1. 아래 파일들이 **같은 폴더** 안에 있는지 확인합니다.
   ```
   presentation.html   ← 메인 파일
   style.css           ← 디자인 스타일시트
   script.js           ← 슬라이드 동작 엔진
   ```
2. `presentation.html` 파일을 **Chrome, Edge, Safari** 등 최신 웹 브라우저로 엽니다.
3. 슬라이드가 화면을 꽉 채우며 자동으로 시작됩니다.
4. 발표 시에는 `F` 키를 눌러 **전체화면 모드**로 전환하는 것을 권장합니다.

> ⚠️ 인터넷 연결 없이 로컬에서 실행 가능합니다. 단, 파일 3개가 반드시 같은 폴더에 위치해야 합니다.

---

## ⌨️ 조작 방법 (단축키)

| 키 / 버튼 | 동작 |
|---|---|
| `→` 방향키 또는 `Space` | 다음 슬라이드로 이동 |
| `←` 방향키 | 이전 슬라이드로 이동 |
| `Tab` | 전체 슬라이드 개요(Overview) 열기 / 닫기 |
| `F` | 전체화면 전환 / 해제 |

화면 **좌측 하단**에 위치한 `Prev` / `Overview (Tab)` / `Next` 버튼을 클릭해도 동일한 기능을 사용할 수 있습니다.  
마우스 조작만으로도 발표 진행이 가능하므로 상황에 따라 선택하여 사용하세요.

## 🖥️ 화면 구성 요소

### 1. 상단 진행 바 (Progress Bar)
화면 최상단에 가느다란 바가 표시됩니다. 슬라이드가 넘어갈 때마다 초록색 그라데이션 바가 오른쪽으로 늘어나며, 전체 발표에서 현재 위치가 어느 정도인지 시각적으로 파악할 수 있습니다.

### 2. 슬라이드 번호 인덱스
각 슬라이드 우측 상단에 반투명하게 표시되는 대형 숫자(01 ~ 15)입니다. 배경 장식 역할을 하며, 현재 슬라이드 번호를 직관적으로 인식할 수 있게 도와줍니다.

### 3. 글래스 카드 (Glass Card)
본문 내용을 담는 반투명 유리 질감의 카드 컴포넌트입니다. 마우스를 올리면 카드가 위로 살짝 떠오르며 테두리가 청록색(#00ffe0)으로 발광하는 hover 효과가 적용됩니다.

### 4. 파티클 배경 네트워크
화면 전체에 80개의 미세한 발광 파티클이 느리게 부유하며 SF 분위기의 배경을 형성합니다. 화면 크기가 바뀌어도 자동으로 캔버스 크기를 재조정합니다.

### 5. 하단 힌트 바
화면 우측 하단에 `← → 방향키 이동 | TAB 개요 모드 | F 전체화면` 문구가 상시 표시되어 조작법을 빠르게 확인할 수 있습니다.

---

## 🗂️ 개요 모드 (Overview Mode)

`Tab` 키 또는 `Overview (Tab)` 버튼을 누르면 전체 슬라이드가 **썸네일 그리드 형태**로 한 화면에 표시됩니다.

- 현재 위치한 슬라이드는 청록색 테두리로 강조 표시됩니다.
- 원하는 썸네일을 클릭하면 해당 슬라이드로 **즉시 이동**하고 개요 화면이 자동으로 닫힙니다.
- 발표 중간에 특정 슬라이드로 빠르게 넘어가야 할 때 유용합니다.
- 다시 `Tab`을 누르면 개요 모드가 닫히고 현재 슬라이드로 돌아옵니다.

## ✨ 특수 효과

### 타이핑 애니메이션
첫 번째 슬라이드(01. VIBE CODING)에 진입하면 터미널 창 안의 텍스트가 **한 글자씩 타이핑되는 효과**로 출력됩니다. 한 번 완료된 이후에는 다시 재생되지 않습니다.

### 슬라이드 전환 효과
슬라이드 이동 시 수평 슬라이딩 애니메이션이 적용되며, 비활성 슬라이드는 투명도와 크기가 약간 줄어든 상태로 유지됩니다. 활성 슬라이드는 완전한 불투명도와 원래 크기로 복원되어 포커스 효과를 만들어냅니다.
